# 双源仲裁规则

## 字段级比对（OpenAlex × Crossref）

按以下字段独立比对，分别打标，最后合成总状态。

| 字段 | 比对方式 | 容忍度 |
|---|---|---|
| 标题 | 归一化后字符串相似度（Levenshtein 或简单字符重合度） | ≥0.95 一致；0.80–0.95 弱一致 + 标注差异；<0.80 冲突 |
| 第一作者 | 归一化姓名比对 | 完全匹配 = 一致；拼音 vs 汉字 = 弱一致（不视为冲突，但降级标注） |
| 期刊 | 名称归一化 + 中英文互译识别 | 一致 / 弱一致 / 冲突 |
| 年 | 数值 | 一致 / 冲突（差 ≥1 即冲突，不允许"约等于"） |
| 卷 / 期 / 页 | 数值或字符串 | 一致 / 冲突；任一源缺该字段 = 跳过此字段（不算冲突） |
| DOI | 大小写无关字符串 | 一致 / 冲突；DOI 冲突是硬冲突（不弱一致） |

## 中文标题归一化

按顺序应用：

1. 繁体 → 简体（用 OpenCC 思路，无实装时退化为常见 1:1 映射）
2. 全角 → 半角（数字 0-9、字母 A-Z/a-z、常用标点）
3. 中文标点统一：
   - 保留：《》 "" '' 「」 『』
   - 统一为半角：，。、；：？！（） 等
4. 折叠多空格 → 单空格
5. 去前后空白
6. 英文部分大小写无关

## 中文期刊名归一化

中文期刊在 OpenAlex 与 Crossref 常出现以下情况：

| OpenAlex | Crossref | 判断 |
|---|---|---|
| 中文官名 | 中文官名 | 标题归一化后比 |
| 中文官名 | 英文官名 | **弱一致** + 标注"中英文互译，可能为同一刊" |
| 英文官名 | 中文官名 | 同上 |
| 一边给缩写一边给全称 | — | 弱一致 + 标注 |

弱一致**不阻塞**"已核验·双源"判定，但报告中需附说明。

## 中文姓名归一化

| 形式 | 处理 |
|---|---|
| 汉字（如"张三"） | 标准形式 |
| 拼音首字母（如"Z. San"） | 还原难，标注为"拼音缩写，可能匹配" |
| 全拼音（如"Zhang San"） | 与汉字版比对 → 弱一致 |
| 拼音 family + given（如 `family=Zhang, given=San`） | 合成 "Zhang San" 后同上 |

**禁止**：用拼音猜汉字、用汉字猜拼音、用记忆里的"知名学者"补全。

## 仲裁结论矩阵

| OpenAlex | Crossref | 字段比对 | 结论 |
|---|---|---|---|
| 命中 | 命中 | 全字段一致 | **已核验·双源** |
| 命中 | 命中 | 1–2 字段弱一致，其余一致 | **已核验·双源**（附弱一致说明） |
| 命中 | 命中 | ≥1 字段硬冲突 | **冲突·{字段}**（列两源原文，不擅断） |
| 命中 | 未命中 | — | **已核验·单源·OA**（低中置信） |
| 未命中 | 命中 | — | **已核验·单源·CR**（中置信，CR 元数据通常更规范） |
| 命中 | error / 不可达 | — | **已核验·单源·OA** + 显式标注"另一源不可用" |
| error / 不可达 | 命中 | — | **已核验·单源·CR** + 显式标注"另一源不可用" |
| 都未命中 | — | — | 走 Step 4 兜底（ChinaXiv 条件触发）；都未触发兜底 → **未核验·信源失联** |
| 两源都 error / 不可达 | — | — | **STOP** — 信源失联即停（不出报告，告知用户后等待） |

## 冲突报告格式

```yaml
状态: 冲突·第一作者
冲突字段:
  第一作者:
    OpenAlex: "王五"
    Crossref: "王五"
    用户原文: "李四"
    判断: 两源一致指向王五，用户原文可能笔误。需人工确认。
其他字段（双源一致部分）:
  标题: <统一值>
  期刊: <统一值或弱一致说明>
  年: <统一值>
  ...
```

## 总状态合成（多字段冲突）

若同时有多个字段冲突，状态写成 `冲突·{字段1}·{字段2}`。
报告中**每个冲突字段都要展开**，不许笼统说"多字段冲突"。

## 边界：双源同时返回但内容空

OpenAlex 偶尔返回 `results: []` 但 HTTP 200。Crossref 类似。
这种情况**视为 not_found**，不视为命中。
