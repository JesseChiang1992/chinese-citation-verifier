# 案例 B：字段冲突（禁止擅断）

## 场景

用户提供的引用作者写错了（实际作者是"王五"，用户写成"李四"），同时 OpenAlex 与 Crossref 两边登记的期刊名一边中文一边英文。

## 输入

```
李四, 赵六. 某中文研究. 中华骨科杂志, 2023, 43(5): 300-308.
```

## 执行流程

### Step 1：归一化
- 无 DOI，提取标题/作者/期刊/年/卷期页作为搜索条件。

### Step 2：查询（并行）

```
WebFetch(OpenAlex, search=某中文研究, filter=author.display_name.search:李四)
WebFetch(Crossref, query.bibliographic="某中文研究 李四 2023")
```

OpenAlex 返回最佳匹配：
- 标题：某中文研究 ✓
- 第一作者：**王五**
- 期刊：中华骨科杂志
- 年/卷/期/页：一致

Crossref 返回最佳匹配（同一 DOI）：
- 标题：某中文研究 ✓
- 第一作者：**王五** (`family=王五, given=`)
- 期刊：**Chinese Journal of Orthopaedics**（英文官名）
- 年/卷/期/页：一致

### Step 3：仲裁

| 字段 | OpenAlex | Crossref | 用户原文 | 判定 |
|---|---|---|---|---|
| 标题 | 某中文研究 | 某中文研究 | 某中文研究 | 一致 |
| 第一作者 | 王五 | 王五 | **李四** | **两源一致但与用户原文冲突** |
| 期刊 | 中华骨科杂志 | Chinese Journal of Orthopaedics | 中华骨科杂志 | 中英文互译，**弱一致** |
| 年 | 2023 | 2023 | 2023 | 一致 |
| 卷/期/页 | 43/5/300-308 | 43/5/300-308 | 43/5/300-308 | 一致 |

→ 状态：**冲突·第一作者**（用户原文与双源不符；期刊为弱一致，附说明）

### Step 4-5：跳过

### Step 6：出报告

## 输出

```yaml
原始引用: 李四, 赵六. 某中文研究. 中华骨科杂志, 2023, 43(5): 300-308.
核验状态: 冲突·第一作者
信源命中: [OpenAlex, Crossref]
规范元数据（双源一致部分）:
  标题: 某中文研究
  期刊: "中华骨科杂志" / "Chinese Journal of Orthopaedics"  # 中英文互译，弱一致
  年: 2023
  卷: "43"
  期: "5"
  页: "300-308"
冲突字段:
  第一作者:
    用户原文: 李四
    OpenAlex: 王五
    Crossref: 王五
    判断: 两源一致指向"王五"，用户原文"李四"可能笔误或张冠李戴。需人工确认。
弱一致字段:
  期刊:
    OpenAlex: 中华骨科杂志
    Crossref: Chinese Journal of Orthopaedics
    判断: 同一期刊的中英文官名，视为弱一致（不冲突）
摘要相关度: 未做此项
```

## 要点（禁令）

- **禁止**自行替换"李四"→"王五"输出已核验报告。即使两源同意，也必须报告冲突让人决。
- **禁止**笼统说"作者不对"——必须列出三方原文：用户原文、OpenAlex、Crossref。
- 期刊中英文互译是**弱一致**而非冲突——不必上报为冲突字段，但需在报告中标注。
- 若用户确认"是笔误，王五对"，**那是下一步用户的决策**，不在本 skill 范围内自动修。
