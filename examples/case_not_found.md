# 案例 C：信源失联 / 未命中（诚信门退出）

## 场景

一篇较老的地方期刊文章，无 DOI，OpenAlex 未收录，Crossref 上对应期刊未注册 DOI。

## 输入

```
钱七. 某地方医学杂志一篇文. 某省医学杂志, 2008, 12(3): 45-47.
```

## 执行流程

### Step 1：归一化
- 提取候选字段（无 DOI）

### Step 2：查询（并行）

```
WebFetch(OpenAlex, search=某地方医学杂志一篇文, filter=author.display_name.search:钱七)
  → meta.count == 0 → OA: no_match

WebFetch(Crossref, query.bibliographic="某地方医学杂志一篇文 钱七 2008")
  → message.items == [] → CR: not_found
```

### Step 3：仲裁
- 两源都未命中 → 进入 Step 4 兜底判断

### Step 4：兜底
- 引用文本无「预印本 / preprint / chinaxiv」字样 → **跳过 ChinaXiv**
- → 直接进 Step 6 出未核验报告

### Step 5：跳过

### Step 6：出报告

## 输出

```yaml
原始引用: 钱七. 某地方医学杂志一篇文. 某省医学杂志, 2008, 12(3): 45-47.
核验状态: 未核验·信源失联
信源命中: []
失联原因:
  OpenAlex: no_match
  Crossref: not_found
  ChinaXiv: 未触发（引用文本无预印本线索）
规范元数据: 不补全
  # 按"查不到不补"铁律 —— 不凭记忆补全任何字段
  # 用户原文中给出的字段维持原状（用户提供，未经本 skill 验证）
建议:
  - 用户可手动到 CNKI / 万方 / SinoMed 检索后回传结果
  - 检查期刊名是否准确（"某省医学杂志" 可能是非正式名 / 已停刊）
  - 检查作者姓名拼写
  - 若可获取原文 PDF，可手动提交摘要做相关度判断
摘要相关度: 摘要不可得（未做此项）
```

## 要点（铁律）

- **禁止**凭记忆"补"标题、DOI、卷期页等任何字段。
- **禁止**把"未核验"包装成"已核验"。
- **禁止**不报告失联就跳过此条 —— 每条都必须有显性标注。
- 用户原文中的字段维持原样，但**不打"已核验"标签**。
- 给出**可操作的下一步建议**（人工检索通道），不只是说"查不到"。

## 极端边界：两源都 error（非 not_found）

若 OpenAlex 与 Crossref 同时**网络不可达 / HTTP 5xx**（非"查得到但没数据"，而是"根本查不动"）：

- **STOP** — 信源失联即停
- **不出报告**，告知用户网络/真相源不可用
- 等待用户确认（重试 / 跳过 / 中止）
- **绝不**用记忆顶替继续输出

```yaml
执行状态: STOP - 信源失联
失联类型: 两源同时不可达（OpenAlex error + Crossref error）
建议: 等待网络恢复后重试；或人工核验
```
