# 摘要相关度判断规则

## 启动条件

**仅当**用户原任务中包含「这篇文献**支持** / **反对** / **证明** 论点 X」类声明时启动。
否则**跳过**摘要相关度，只做元数据核验。

启动信号示例（用户原话）：
- "我引用这篇 [文献] 来支持 [论点 X]"
- "这篇文献是不是真的说了 [论点]"
- "帮我看看这条引用是不是张冠李戴"
- "声明级核查"

非启动情景（仅核元数据）：
- "帮我核对一下这几条引用"（无具体论点）
- "补全这条文献的元数据"
- "这条 DOI 真的存在吗"

## 摘要获取顺序

按优先级取摘要（任一拿到即停）：

1. **OpenAlex** `abstract_inverted_index` 重构（见 `REFERENCE_OpenAlex.md` 摘要重构段）
2. **Crossref** `abstract` 字段（清洗 JATS 标签）
3. **ChinaXiv** 详情页摘要段（仅当 ChinaXiv 兜底已触发）
4. 都没有 → 标 `摘要不可得`，**禁止**编造摘要

## 判断分级

| 分级 | 含义 | 锚点要求 |
|---|---|---|
| 支持 | 摘要明确表述了论点 X 的核心结论 | **必须**引用摘要中至少 1 句原文 |
| 部分支持 | 摘要涉及论点 X 的部分但有保留 / 限定 / 仅子集成立 | **必须**引用关键限定句 + 说明限定范围 |
| 不支持 | 摘要主题与论点 X 无关 / 反向 / 张冠李戴 | **必须**说明哪里不对应 |
| 摘要不可得 | 真相源都未给摘要 | 退出相关度判断，不强行 |

## 禁令

- 禁止"读完之后我感觉"式判断 —— 必须**回到摘要原文**引片段。
- 摘要不可得就是不可得，**不许**用标题 / 期刊 / 作者推断摘要内容。
- 摘要内容若与用户论点冲突，**必须如实指出**（不为讨好用户而修饰为"部分支持"）。
- 禁止把"标题相关"包装成"摘要支持"——标题相关 ≠ 摘要支持。
- 单一摘要不足以判断时，标"部分支持"或"摘要不可得"，**不许**强行升格为"支持"。

## 输出格式

```yaml
论点: <用户给出的论点 X，逐字引用>
摘要来源: OpenAlex | Crossref | ChinaXiv
摘要片段（锚点）: "<原文 1-2 句，截至 200 字>"
相关度: 支持 | 部分支持 | 不支持
理由: <一句话说明锚点如何对应论点>
风险: <可选；如"仅摘要可见，全文未核"等限定>
```

## 与元数据核验的串联

摘要相关度是**Step 5（条件可选）**，前置依赖：
- Step 2 已拿到 OpenAlex / Crossref / ChinaXiv 至少一源的命中结果
- 命中结果中含摘要（OpenAlex 通常有，Crossref 视出版商）

若元数据核验得到"未核验·信源失联"，**不可能**有摘要可查 —— 摘要相关度自动标 `摘要不可得`，整条引用打 `未核验·信源失联`。

## 与全文核查的边界

本 skill **只看摘要**。若用户希望声明级核查需要全文（例如摘要中无明确表述、需要查 Methods/Results），应：
- 标 `摘要不足以判断，建议核全文`
- **不调用**全文获取（v1 不接入 PDF 解析）
- 建议用户走 `mineru-pdf-parser` + 人工提供 PDF 后再做声明级核查
